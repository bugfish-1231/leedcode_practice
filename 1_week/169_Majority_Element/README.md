### 📝 題目：[ 0169. Majority Element ]

#### 🧠 解題思路
這題的目標是找出陣列中出現次數大於 ⌊n / 2⌋ 的元素。題目保證這個元素一定存在。
1. **直覺解法 (Hash Map)**
   - 思路：延續上一題的經驗，我們可以用 unordered_map<int, int> 來紀錄 [數字, 出現次數]。把陣列掃過一次，誰的次數先超過 n / 2，誰就是答案。
   - 時間複雜度：$O(n)$。遍歷陣列一次。
   - 空間複雜度：$O(n)$。最壞情況下需要存很多不同的數字。
2. **聰明解法 (Sorting 排序法)**
   - 思路：既然這個「多數元素」的數量超過了整個陣列的一半，那麼如果我們把陣列從小到大排好序，陣列最中間的那個數字 nums[n / 2]，絕對會是這個多數元素！
   - 時間複雜度：$O(n \log n)$。C++ 內建排序 std::sort 的速度。
   - 空間複雜度：$O(1)$ 或 $O(\log n)$（取決於 C++ 底層排序演算法的空間消耗）。
3. **最佳化解法 (Boyer-Moore Voting Algorithm 投票演算法)**
   - 思路：這是面試官最想看到的解法。把它想像成「打群架」或「投票」。
   因為多數派的人數大於其他所有人數的總和，我們只要設定一個 candidate (候選人) 和一個 count (票數)。
    - 遇到同陣營的人，票數 +1。
    - 遇到不同陣營的人，票數 -1 (互相抵銷)。
    - 當票數歸零時，代表前面的人互相死光了，我們把當前遇到的人設為新的 candidate。
    - 戰到最後，因為多數派人數最多，絕對會有剩下來的人，那個人就是答案。
   - 時間複雜度：$O(n)$。只要掃描陣列一次。
   - 空間複雜度：$O(1)$。只用了兩個變數，完全不耗費額外記憶體。

#### 💻 語言差異與踩坑紀錄
* **C++**：
  * 學到了什麼：
    1. 演算法的降維打擊：有時候不需要真實去「計數」，利用題目「超過一半」的數學特性就能找出 $O(1)$ 空間的最佳解。
    2. 如果想用排序法，學會了 C++ 內建的排序函數 sort(nums.begin(), nums.end()); 的用法。

#### 🏆 總結 (Takeaway)
* 這題考驗的核心盲點是什麼？ 很容易陷入「必須精確算出每個數字出現幾次」的思維陷阱。
* 下次看到類似題型，第一步該想到什麼工具？
只要題目關鍵字出現**「超過一半 (Majority)」**，請在大腦中反射性地閃過 Boyer-Moore Voting Algorithm (投票法)。
